# 🏥 Clinic Queue & Ticket Management System

## 📋 Project Overview
This project is a Queue & Ticket Management System designed to manage patient flow in clinics efficiently.

The system was originally developed using PHP and runs on a local XAMPP environment. It has been previously modified to improve UI and functionality. However, it still requires further refactoring to improve performance, security, and code structure.

The goal of this project is to **rebuild and enhance the system using clean and professional PHP architecture** while maintaining the original workflow.

## 🎯 Project Objectives

- Develop an efficient and structured queue management system
- Improve patient flow and reduce waiting time
- Enhance operational efficiency in clinics and medical centers
- Provide a simple and user-friendly interface for staff and patients
- Ensure system stability, scalability, and maintainability
- Apply secure coding practices and performance optimization

## ⚙️ System Features

- ✅ Patient ticket generation (Queue system)
- ✅ Clinic and specialization selection
- ✅ Real-time waiting display system
- ✅ Doctor dashboard for patient management
- ✅ Next patient calling system
- ✅ Emergency case prioritization
- ✅ Patient transfer between doctors or clinics
- ✅ Configurable system for different medical environments

## 🛠 Technologies Used

- Frontend: HTML5, CSS3, Bootstrap, JavaScript
- Backend: PHP
- Database: MySQL
- Environment: XAMPP (Local Development)

## 🧩 System Modules

### 👤 Patient Module
- Select clinic or specialization
- Get queue number
- View waiting status

### 🖥 Display Screen Module
- Shows current patient number
- Displays clinic and room information
- Real-time updates for queue

### 👨‍⚕️ Doctor Dashboard
- Call next patient
- Mark emergency cases
- Transfer patient to another clinic/doctor
- Manage queue workflow
